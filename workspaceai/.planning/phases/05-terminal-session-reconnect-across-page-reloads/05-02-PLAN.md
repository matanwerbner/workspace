<objective>
Make the renderer reconnect to an existing pty instead of orphaning it on a Vite full page reload. Extract the async pty-wiring logic currently buried in an IIFE inside `getOrCreateSession` into a pure, dependency-injected `attachSession(deps)` orchestrator in `src/views/terminal/attachSession.ts`, implement the reconnect-first-then-create flow (replaying the buffered scrollback on reconnect, forwarding `instanceId` as `viewId` to create so the new pty is registered for next time), and wire `getOrCreateSession` to call it with real dependencies. Cover the orchestrator with node-environment unit tests.

Purpose: This is the consumer side of the contract built in plan 05-01. It delivers TERM-03: before spawning a new pty, the renderer asks `terminalReconnect(instanceId)`; on a hit it adopts the live termId and replays the up-to-50 KB buffer so the user's shell continues exactly where it left off; on a miss it falls through to `terminalCreate`. Extracting `attachSession` makes this decision logic unit-testable in the existing node test environment (no jsdom, no real xterm), which is required for Nyquist verification since the reconnect branch is the whole point of the phase.

Output:
- `src/views/terminal/attachSession.ts` — `attachSession(deps)` orchestrator + its `AttachDeps` interface
- `src/views/terminal/__tests__/attachSession.test.ts` — reconnect, create-fallback, and disposed-before-resolve tests
- `src/views/terminal/sessionCache.ts` — `getOrCreateSession` delegates to `attachSession`
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main → renderer (replayed buffer) | The outputBuf returned by terminal:reconnect is written directly into the xterm terminal |
| renderer session lifecycle | A session may be disposed (view removed) between the reconnect/create request and its resolution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | replayed outputBuf containing terminal escape sequences written into xterm | accept | the buffer is the user's own captured shell output (already rendered live once before the reload); xterm is the same renderer that produced it. No new trust boundary — this is identical to the live data path that already writes raw pty bytes into terminal.write(). |
| T-05-06 | Resource leak | a pty spawned by create after the view was disposed mid-flight | mitigate | attachSession checks deps.isDisposed() after create resolves and calls deps.kill(termId) — preserves the existing orphan-kill guard from the old IIFE (sessionCache.ts:95-98) |
| T-05-07 | Tampering | wiring listeners against a stale/disposed session | mitigate | onResolved/wireListeners only run when isDisposed() is false; a disposed reconnect target is left untouched (the registry/disposeSession owns its kill) |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this phase — slopcheck N/A; no install tasks exist |
</threat_model>
