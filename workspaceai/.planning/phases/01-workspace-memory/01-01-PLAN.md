<artifacts_this_phase_produces>
This plan introduces the following symbols (consumed by plan 01-03):

**Pure helpers — `src/shell/memory.ts`:**
- `isValidMemoryName(name: string): boolean` — true only for `/^[a-z0-9][a-z0-9-]*$/` (no `.`, no `/`, no `..`)
- `formatEntry(name: string, description: string, type: string, content: string): string` — returns the full `.md` file body with YAML frontmatter (`name`, `description`, `metadata.type`) + blank line + content
- `upsertIndexLine(existingIndex: string, name: string, description: string, fileName: string): string` — parses existing `- [...]` lines, removes any prior line for `name`, appends the canonical line, returns the regenerated index text ending in `\n`
- `buildMemorySection(indexContent: string | null): string` — returns the `## Workspace Memory` system-prompt section, or `''` when index is null/blank (consumed by plan 01-03)
- `MEMORY_TYPES` — readonly tuple `['user', 'project', 'feedback', 'reference']`

**IPC channels — `electron/ipc/memory.ts`:**
- `memory:readIndex(homeFolder: string): Promise<string | null>`
- `memory:readEntry(homeFolder: string, topic: string): Promise<string | null>`
- `memory:writeEntry(homeFolder: string, name: string, description: string, type: string, content: string): Promise<void>`
- `registerMemoryHandlers(): void`
</artifacts_this_phase_produces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (IPC) | Renderer supplies homeFolder, name, topic, content as untrusted IPC arguments |
| main → filesystem | Memory paths cross into real disk writes/reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | memory:writeEntry `name` param (path traversal e.g. `../../etc/passwd`) | mitigate | `isValidMemoryName` rejects anything not matching `/^[a-z0-9][a-z0-9-]*$/` before path construction; `assertParentInsideRoots` is a second layer (per 01-RESEARCH Security Domain) |
| T-01-02 | Elevation of privilege | symlink at `{homeFolder}/memory/{name}.md` pointing outside root | mitigate | `lstat().isSymbolicLink()` check before write, copied from fs.ts:53-59; reject with 'Access denied' |
| T-01-03 | Tampering | renderer-supplied `homeFolder` pointing outside user dirs | mitigate | `assertInsideRoots`/`assertParentInsideRoots` only pass when homeFolder is a registered root (registration handled in plan 01-02); unregistered paths throw/return null |
| T-01-04 | Tampering | `type` param outside the four allowed values | mitigate | writeEntry throws 'Invalid memory type' unless `MEMORY_TYPES.includes(type)` |
| T-01-05 | Information disclosure | large memory bodies written verbatim into session logs | mitigate | `memory:writeEntry` added to SUMMARIZED_CHANNELS; only { name, type, bytes } logged |
</threat_model>
