<objective>
Close the critical security/functionality gap: `workspace.homeFolder` is never registered as a trusted root, so every `memory:*` IPC call would fail the `assertInsideRoots` path check. Patch the two registration points — startup state seeding and active-workspace switching — so the homeFolder becomes a registered root, and add a unit test that proves homeFolder strings are picked up from seeded state.

Purpose: Without this, the memory storage layer built in plan 01-01 is unusable — every read/write throws "Access denied: path is outside the workspace root". This is the single highest-risk finding in 01-RESEARCH.md (Finding 5, Pitfall 1).

Output:
- `electron/ipc/roots.ts` — `seedRootsFromState` patched to register each `workspace.homeFolder`, with the homeFolder-collection logic exposed in a way that is unit-testable from `src/`
- `electron/ipc/workspace.ts` — `workspace:setActiveHomeFolder` patched to call `registerRoot(path)` when a homeFolder is set
- `src/state/__tests__/seedRoots.test.ts` — proves the collection logic picks up homeFolder strings
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted state → roots allowlist | Startup seeds trusted roots from on-disk state |
| renderer → main (setActiveHomeFolder) | Renderer supplies the active homeFolder path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Elevation of privilege | registering an attacker-controlled homeFolder as a root | mitigate | registerRoot calls realpathSync + statSync().isDirectory() and only adds existing real directories; a non-existent or non-dir path is silently ignored (roots.ts:13-23, unchanged) |
| T-01-07 | Tampering | malformed/partial persisted state crashing seedRootsFromState | accept→mitigate | collectRootPaths preserves the existing typeof/Array.isArray guards, returns [] on malformed input rather than throwing |
| T-01-08 | Spoofing | renderer passing a homeFolder it does not own | accept | homeFolder originates from persisted workspace state the user themselves created via the folder picker; registerRoot still requires the path to be a real existing directory. Low risk in a single-user desktop app |
</threat_model>
