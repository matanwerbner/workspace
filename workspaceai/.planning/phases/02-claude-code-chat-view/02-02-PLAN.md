| Boundary | Description |
|----------|-------------|
| AI response → renderMarkdown → dangerouslySetInnerHTML | Untrusted assistant text flows through renderMarkdown (XSS mitigation from Plan 01) into the DOM |
| User-chosen view type → view registry | typeId string from store must match a registered view; unknown typeIds are handled by the default branch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | XSS | ClaudeCodeView dangerouslySetInnerHTML | mitigate | renderMarkdown escapes raw HTML (Renderer.html override from Plan 01); hljs output only contains safe span elements |
| T-02-05 | Spoofing | buildSystemPrompt default branch | accept | Unknown typeIds fall through to default; no crash, no leak |
| T-02-06 | Information Disclosure | selectChatMessages in view layer | accept | Messages are already in client memory (Zustand store); no new data crosses trust boundary |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new installs in this plan; highlight.js already installed and verified in Plan 01 |
</threat_model>
