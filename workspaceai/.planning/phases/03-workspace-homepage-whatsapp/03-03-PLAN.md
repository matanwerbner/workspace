       Imports:
       - ipcMain, BrowserWindow, app from 'electron'
       - join from 'node:path'
       - makeClient from './ai'
       - Types: WhatsAppAgentStatus, WhatsAppStatusEvent, WhatsAppMessageEvent from '../../src/state/types'
       - Anthropic from '@anthropic-ai/sdk' (for MessageParam type only)
       - Baileys: import makeWASocket, DisconnectReason, useMultiFileAuthState from '@whiskeysockets/baileys'
         (Use the Baileys v6.7.x API surface; check the Baileys README for the exact named export spellings if import fails)

       Module-level singleton state (follow terminal.ts pattern):
       - let sock: ReturnType<typeof makeWASocket> | null = null
       - let agentStatus: WhatsAppAgentStatus = { state: 'disconnected' }
       - const conversationHistory = new Map<string, Anthropic.MessageParam[]>()
       - const MAX_HISTORY_TURNS = 20  (cap per JID)
       - const authDir = computed once from join(app.getPath('userData'), 'whatsapp-auth')

       Helper: pushStatus(payload: WhatsAppStatusEvent): void
       - BrowserWindow.getAllWindows()[0]?.webContents.send('whatsapp:status', payload)

       Helper: pushMessage(payload: WhatsAppMessageEvent): void
       - BrowserWindow.getAllWindows()[0]?.webContents.send('whatsapp:message', payload)

       Helper: runAgentTurn(userText: string, jid: string): Promise<string>
       - Call makeClient(); if null return '(No API key configured)'
       - Get existing history from conversationHistory.get(jid) ?? []
       - Build messages array: [...history.slice(-(MAX_HISTORY_TURNS * 2)), { role: 'user', content: userText }]
         (each turn = 2 entries: user + assistant; multiply by 2 to get turns)
       - Call client.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 1024, messages })
       - Find first text block in result.content
       - Update history: conversationHistory.set(jid, [...messages, { role: 'assistant', content: reply }])
       - Return reply string (or '' if no text block)
       - Wrap in try/catch; on error update agentStatus.lastError and re-throw

       export function disposeWhatsApp(): void
       - sock?.end(undefined)
       - sock = null
       - agentStatus = { state: 'disconnected' }
       - conversationHistory.clear()

       export function registerWhatsAppHandlers(): void
       Register these three ipcMain.handle channels:

       'whatsapp:getStatus': returns agentStatus (synchronous)

       'whatsapp:stop': calls disposeWhatsApp(), updates agentStatus = { state: 'disconnected' },
         calls pushStatus({ status: agentStatus })

       'whatsapp:start': async handler
         - If sock is not null, return early (already connected)
         - Set agentStatus = { state: 'connecting' }; pushStatus(...)
         - Await useMultiFileAuthState(authDir) to get { state: authState, saveCreds }
         - Call makeWASocket({ auth: authState, printQRInTerminal: false })
           (printQRInTerminal: false — we push QR via IPC instead)
         - Assign to sock
         - Register sock.ev.on('creds.update', saveCreds)
         - Register sock.ev.on('connection.update', (update) => { ... })
           In the update handler:
             If update.qr exists: set agentStatus = { state: 'qr_pending', qrCode: update.qr }; pushStatus(...)
             If update.connection === 'open': set agentStatus = { state: 'connected' }; pushStatus(...)
             If update.connection === 'close':
               Check update.lastDisconnect?.error?.output?.statusCode against DisconnectReason.loggedOut
               If loggedOut: set agentStatus = { state: 'disconnected' }; sock = null; pushStatus(...)
               Otherwise: reconnect by calling the start handler logic again (schedule with setTimeout 3000)
         - Register sock.ev.on('messages.upsert', async ({ messages: msgs, type }) => { ... })
           Only process if type === 'notify'
           For each msg in msgs where !msg.key.fromMe:
             Extract jid = msg.key.remoteJid
             Extract text from msg.message?.conversation ?? msg.message?.extendedTextMessage?.text ?? ''
             If text is empty, skip
             Push pushMessage({ jid, text, reply: '' }) immediately (for UI display)
             Call reply = await runAgentTurn(text, jid)
             Call await sock.sendMessage(jid, { text: reply })
             Update agentStatus.lastReplyAt = Date.now(); pushStatus(...)
             Push pushMessage({ jid, text, reply })

       Security constraints (per RESEARCH.md Security Domain):
       - Agent replies ONLY to jid that sent the message (no broadcasting)
       - No tool calls in MVP — single-turn text only
       - Auth state files stored in userData/whatsapp-auth/ (NOT in project dir)
       - Wrap all ipcMain.handle callbacks in try/catch; throw new Error(e instanceof Error ? e.message : String(e))

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhatsApp → main process | Incoming message text from external WhatsApp senders |
| main process → Claude API | Message text forwarded to Anthropic; no sanitization needed since it's not executed |
| main process → WhatsApp | Claude reply sent back to the originating JID only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-PI | Tampering | runAgentTurn — incoming WA message text | mitigate | Text is passed verbatim to Claude as user content only; no tool calls in MVP; system prompt (if added) should include role-lock instruction |
| T-03-AC | Information Disclosure | Agent replies only to sender JID | mitigate | sock.sendMessage(jid, ...) where jid === msg.key.remoteJid; no broadcast path |
| T-03-AS | Denial of Service | Runaway message processing | mitigate | Single-turn (no agentic loop); no per-message rate limit in MVP — acceptable for personal account scope |
| T-03-AU | Information Disclosure | Baileys auth state files | mitigate | Stored in app.getPath('userData')/whatsapp-auth/ — not in project dir, not in homeFolder; Vite watcher does not observe userData |
| T-03-SC | Tampering | npm/cargo install | mitigate | slopcheck + blocking human checkpoint in Plan 03-01 |
</threat_model>
