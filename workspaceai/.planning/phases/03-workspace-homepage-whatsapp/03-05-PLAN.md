<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (invoke) | whatsappStart/Stop/GetStatus: renderer-initiated lifecycle commands |
| main → renderer (push) | whatsapp:status and whatsapp:message: unsolicited events pushed to renderer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-W1 | Elevation of Privilege | whatsapp:start IPC handler | accept | Renderer can only trigger connect/disconnect; main process controls all socket state; no renderer-supplied data reaches Baileys except the start/stop command |
| T-03-W2 | Denial of Service | Rapid start/stop from renderer | mitigate | whatsapp:start is a no-op if sock is already connected (early return guard in Plan 03-03) |
| T-03-W3 | Information Disclosure | whatsapp:status push events contain qrCode | accept | Events delivered only to the Electron renderer window; contextBridge prevents renderer access to raw ipcRenderer — push payloads pass through the typed preload bridge |
| T-03-SC | Tampering | npm/cargo installs | mitigate | slopcheck + blocking human checkpoint in Plan 03-01 |
</threat_model>
